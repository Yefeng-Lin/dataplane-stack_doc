This chapter provides the design specification for a user case/feature
provided by the software. Each use case design specification may include
information as:

- Explain the use case.
- Show in diagram the components involved.
- Show a diagram outlining any important data/control flows through the
  components.
- Explain any test steps that exist for the use case.

And the key components constituting the solution could also be described in
details with information as:

- Explain how the solution is integrated with which components and which
  tools are involved (e.g.: DPDK, VPP)
- Show in diagram the components/projects/tools involved and its dependencies.
